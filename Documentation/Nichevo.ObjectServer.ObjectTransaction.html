Remarks

An ObjectTransaction allows actions such has

  • Create - Initialise a new object with default values and a generated primary key
  • Select - Initialise a new object or collection of objects with values from the data source
  • Delete - Deleting existing objects from the data source
  • Commit - Updating the data source with the operations performed through the transaction
An ObjectTransaction is created by an ObjectManager, and each ObjectTransaction has its own dedicated connection to the data source.

Each ObjectTransaction maintains a cache of the objects loaded so far and trys to resolve references by returning cached objects first. This means that the ObjectTransaction maintains object identity within the scope of the transaction. This allows object identity to be compared simple on object reference equality.
