Remarks

Parameter names should follow the SQL guidelines "@id" etc. The only restrictionis that parameters should NOT be named the same as a property they will be contraining. For instance "Name = @Name" is not allowed due to the parameter name equalling a property name, "Name = @name" is allowed due to parameter names in Contraint being case senstive.

This IS case senstive.
